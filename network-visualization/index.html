Network Visualization Project Log: 1

June 17, 2015
Intro

I am working on a network visualization web application. The purpose is to visual the network during cyber defense competitions. I am working with someone else. I am working on the server-side of the web app and setting up the testing infrastructure. My partner will be working on the client-side making the graphics using webGL. This project is suppose to be similar to Norse's map.ipviking.com but for a local network and will show the different teams instead of continents.

Services Used:
  • Argus
  • Node.js
  • Socket.io

The test environment will be on a CentOS 6.6 Virtual Machine. For now I will install everything on the same system.

Argus is a network monitoring tool that will used to capture and produce the network flow in real time. Node.js will be used with socket.io to make a web socket that will feed the client(s) information.

Current Status:
  • The Argus server and client is working.
  • Working on getting a demo up that will show the argus client information on a web page.
  • Writing the Node.js server that will output the data.
